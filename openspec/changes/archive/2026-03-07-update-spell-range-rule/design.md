## Context

目前專案中透過 `app.py` 處理 Google Document 的文字擷取，其中對於「施法距離」的判斷規則在先前的實作或規劃中可能有不同的標準。根據最新的需求 (`需求說明.md`)，施法距離的擷取需要遵循一套精確的字串比對與長度判斷邏輯，以確保填入 Google Sheet 的資料符合預期。

## Goals / Non-Goals

**Goals:**
* 實作精確的「施法距離」擷取邏輯：
  * 開頭符合「近距」、「中距」、「遠距」或「接觸」者，回傳該兩個字。
  * 若不符合上述條件，且字串長度小於等於 4，則原樣回傳。
  * 其他情況一律回傳「其他」。
* 確保這個更新的邏輯能無縫整合進目前的文字處理流程中。

**Non-Goals:**
* 修改其他法術屬性（如成分、時間、持續時間、豁免等）的擷取規則，這些部分保持其既定或先前的修改邏輯。
* 更動 Google API 的連線或讀寫架構。

## Decisions

1. **更新 `app.py` 中的解析函式**：
   - 定義一個獨立的距離解析邏輯（可能在 `extract_spell_data` 中，或獨立為 `parse_range(text)` 函式以便於測試與維護）。
   - 使用 `startswith` 檢查特定開頭字串："近距", "中距", "遠距", "接觸"。
   - 移除字串前後的空白後再計算長度，避免因為多餘的空白導致長度判斷錯誤。
   - 使用標準的 `if-elif-else` 結構實作這三個條件分支。

2. **確保相容於現有流程**：
   - 擷取出來的結果同樣需要經過後續的繁簡轉換以及括號替換（如果距離字串中包含括號的情況下）。

## Risks / Trade-offs

* **Risk**: 原始文件中有排版錯誤或隱含不可見字元，導致 `startswith` 判斷失敗或長度計算錯誤。
  * **Mitigation**: 在進行條件判斷前，強制對輸入字串執行 `.strip()` 以移除前後空白及換行符號。
